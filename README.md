### 📌 Gitflow 브랜치 전략

우리 팀은 **Gitflow** 워크플로우를 도입하여 효과적인 협업과 안정적인 코드 관리를 지향합니다. 아래의 브랜치 규칙을 준수하여 개발을 진행해주세요.

* **`main`**: 프로덕션 배포가 가능한 안정적인 코드를 관리합니다. 이 프로젝트는 해당 부분을 release와 통합하여 사용합니다.
* **`develop`**: 다음 배포를 위한 개발이 이루어지는 주 브랜치입니다.
* **`feature/*`**: 새로운 기능 개발을 위한 브랜치입니다. `develop`에서 분기하여 작업 완료 후 `develop`으로 병합합니다. (예: `feature/user-login`, `feature/diary-crud`)
* **`fix/*`**: 버그와 같은 오류 수정을 위한 브랜치입니다. `develop`에서 분기하여 오류 수정작업을 진행합니다.
* **`document/*`**: 문서 작업을 위한 브랜치 입니다. `main`,`develop`에서 분기하여 문서작업을 진행합니다.
* **`hotfix/*`**: `main`에서 발생한 긴급 버그를 수정하는 브랜치입니다. `main`에서 분기하며, 수정 완료 후 `main`과 `develop`에 모두 병합합니다.

### 📚 커밋 메시지 컨벤션

아래의 커밋 메시지 컨벤션을 준수하여 커밋 이력을 명확하게 관리합니다.

| 메시지         | 설명                              |
|:------------|:--------------------------------|
| **`feat`**  | 🥥 새로운 기능 추가 또는 기존 기능 수정        |
| **`fix`**   | 🐛 버그 수정                        |
| **`docs`**  | 📝 문서(주석, README 등) 수정          |
| **`ref`**   | ✏️ 기능 변화가 없는 코드 리팩토링            |
| **`test`**  | ⚙️ 테스트 코드 추가 또는 수정              |
| **`chore`** | 🛠️ 빌드 시스템, 라이브러리, 환경설정 등 기타 수정 |

**💡 커밋 메시지 예시**
* `feat: 로그인 폼 완성`
* `fix: 회원가입 에러 해결`
* `docs: 회원가입 로직 주석 수정`

# 📝 역할분담
## 최정
- 사용자 기본관리(User, Email, Verify): 이 도메인은 회원가입, 로그인, 로그아웃, 회원탈퇴, 개인정보 조회와 같은 사용자의 핵심 계정 기능을 다룹니다. 
특히 JWT를 활용한 토큰 기반 인증 및 인가 시스템을 구현하여 보안을 강화했습니다. 사용자의 이메일은 계정 식별 및 인증의 핵심 요소로 활용됩니다. 
이메일 주소 자체가 별도의 도메인 역할을 하여 사용자 계정에 대한 고유한 인증 정보를 붙일 수 있도록 설계되었습니다. 
계정 상태 관리 기능을 포함하여 사용자 계정의 현재 상태(예: 가입 초기 인증 요청, 인증 완료, 비밀번호 변경 요청, 비밀번호 변경 중)를 효율적으로 추적하고 관리할 수 있습니다.

- 별(Star): 사용자가 일기를 쓰고 별을 획득할 수 있는 기능을 개발하였습니다. 사용자가 자신의 밤하늘에서 별을 자유롭게 관리하고 꾸밀 수 있기에 어플리케이션을 사용하는데 동기부여를 얻습니다.

- 별자리(Constellation): 사용자가 얻은 별을 이용하여 별자리를 제작하고 조회할 수 있는 기능을 개발하였습니다. 웹 브라우저 화면비를 이용한 좌표개념을 도입하였습니다.

- AWS EC2: 서비스의 안정적인 운영을 위한 서버 배포를 담당했습니다. Docker 기반의 개발 환경을 세팅하고, CI/CD 파이프라인을 GitHub Actions를 활용하여 자동화했습니다.

- DB: AWS EC2 환경 내에서 Docker Container로 MySQL 데이터베이스를 관리했습니다. ERD 설계 및 JPA를 활용한 엔티티(Entity) 및 레포지토리(Repository)를 구현하여 데이터의 저장, 조회, 수정, 삭제를 관리했습니다.

- AI : OpenAI API를 활용하여 Moderation과 같은 유해성 검증 서비스와 OpenAI 기본 응답에 프롬프트를 입력하여 저희 서비스에 적용하였습니다.

## 이나현
- 일기(Diary): 어플리케이션의 핵심 기능인 일기 쓰기 기능을 구현하였습니다. 사용자는 감정상태, 감정요인, 일기 제목과 내용을 작성하여 자신의 하루를 기록할 수 있습니다. 
별 색상, 감정, 요인들은 열거형으로 구현하였고, 일기를 작성할 때 StarService에 구현된 별을 생성하는 로직까지 연동하였습니다.

- 마이페이지(All Domains): 마이페이지 기능은 사용자 프로필, 별 개수 기반의 레벨 시스템, 대표 별자리 정보를 한눈에 확인할 수 있도록 통합적으로 제공합니다.
연간 별자리 생성 통계 및 월별 일기 감정 통계를 정확하게 집계하여 사용자의 활동을 심층적으로 분석하고 시각화합니다.
모든 조회 API는 읽기 전용 트랜잭션으로 처리되어 데이터 일관성을 유지하며 닉네임 유효성 검사와 S3 연동을 통한 안전한 프로필 사진 업데이트 기능을 통해 데이터 무결성을 안정적으로 보장합니다.

- 친구(Friend): 친구 기능을 이용하여 다른 사람의 레벨정보를 확인할 수 있도록 구현중입니다.

- AWS S3: 사용자가 자신의 프로필 사진을 업로드할 수 있게 S3에 관현 설정을 담당하였습니다.

- Testing: 지속적인 테스트를 하여 매주 회의에서 발견된 버그들을 보고하였습니다.

- Figma: UI 디자인 작업에 참여하여 프로젝트 진행에 도움을 주었습니다.
